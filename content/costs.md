# Costs

A lightweight spend tracker for the tools behind the build stack.

## What belongs here

- Subscription costs
- One-off credit top-ups
- Spend by account
- Spend by project where relevant
- Notes about why the spend happened

## Review rules

1. Paid spend should map to active work or a clear experiment.
2. If a platform is idle, question the subscription.
3. Compare cost against available free credits before topping up.

## Good questions

- Which platform is costing the most this month?
- Which project is generating the most tool spend?
- Are we paying for overlap between platforms?
